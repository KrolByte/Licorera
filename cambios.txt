LICORERA — DOCUMENTACIÓN DEL PROYECTO

1. Visión general
   - Proyecto: Licorera - sistema de inventario para productos, categorías, proveedores y movimientos.
   - Arquitectura: Desacoplada (Decoupled Application).
   - Frontend: Aplicación estática SPA independiente alojada en Vercel.
   - Backend: API REST construida en Node.js/Express alojada en Render.
   - Base de datos: PostgreSQL administrada en la nube mediante Supabase, utilizando Connection Pooling (IPv4) para optimizar conexiones en entornos cloud.

2. Estructura del proyecto
   - frontend/ (Desplegado de forma independiente)
   - index.html: Interfaz principal y tablas dinámicas del dashboard.
   - styles.css: Estilos visuales adaptativos.
   - app.js: SPA en Vanilla JS; consume los endpoints de la API en producción y renderiza tablas.
   - package.json: Metadatos del frontend (sin dependencias críticas).
   - backend/ (Desplegado de forma independiente)
   - api.js: Servidor Express con endpoints CRUD, habilitación de CORS y lógica de negocio.
   - db.js: Configuración de conexión de base de datos usando el driver pg.
   - package.json: Dependencias de producción (express, pg, cors, dotenv).
   - .env: Almacenamiento seguro de variables de entorno locales (Excluido de GitHub).
   - cambios.txt: Documentación interna del proyecto y registro de notas de cambios.

3. Backend — API y lógica de producción
   - Puerto Dinámico: Configurado mediante variables de entorno (process.env.PORT || 3000) para adaptarse automáticamente a los requerimientos de la infraestructura de Render.
   - Seguridad CORS: Middleware cors() implementado globalmente para permitir solicitudes cruzadas y seguras desde el dominio del Frontend.

   Endpoints principales:
   - Productos: GET /api/productos, GET /api/productos/:id, POST /api/productos, PUT /api/productos/:id, DELETE /api/productos/:id
   - Categorías: GET /api/categorias, POST /api/categorias, PUT /api/categorias/:id, DELETE /api/categorias/:id
   - Proveedores: GET /api/proveedores, POST /api/proveedores, PUT /api/proveedores/:id, DELETE /api/proveedores/:id
   - Movimientos: GET /api/movimientos

   Lógica de inventario:
   - Al crear un producto, se crea un movimiento ENTRADA automático con la cantidad inicial.
   - Al actualizar un producto, se calcula la diferencia de stock y se registra ENTRADA o SALIDA según corresponda.

   Zona horaria:
   - backend/api.js fuerza process.env.TZ = 'America/Bogota' para sincronizar las fechas y horas con Colombia.

4. Frontend — experiencia de usuario y consumo cloud
   - La interfaz consume los datos directamente desde la URL de producción en Render: https://licorera-backend-j97k.onrender.com/api/*.
   - Renderizado dinámico de los componentes <tbody> a través de peticiones asíncronas (fetch).
   - Formularios de creación/edición completamente vinculados para gestionar productos, categorías y proveedores.
   - Se incorpora el contenedor lista-elementos como requerimiento estructural para la visualización de datos.

7. Instrucciones para Ejecución Local (Desarrollo)
   En caso de requerir replicar el proyecto en un entorno local:
   Backend
   - Entrar a la carpeta backend/.
   - Ejecutar npm install.
   - Crear el archivo .env e inyectar la variable DATABASE_URL (usando el formato IPv4 pooler de Supabase).
   - Ejecutar el servidor mediante node api.js o node --watch api.js.
   Frontend
   - Asegurar que la URL del fetch apunte a http://localhost:3000 si se desea desarrollo 100% local, o mantener la URL de Render para pruebas híbridas.
   - Ejecutar el archivo index.html mediante un servidor local como Live Server.
